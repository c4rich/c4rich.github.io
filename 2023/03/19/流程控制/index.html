选择结构

Java中三大结构：顺序结构、选择结构、循环结构

在开发中可能会存在需要对某些条件进行判断的场景。当满足了A条件时，执行A代码，当满足了B条件时，执行B代码…..否则，执行N代码

if单选结构

语法

当满足了条件表达式时（条件表达式的结果为true），就 执行大括号内的代码，否则就跳过大括号内的代码

  1. 判断一个数是否在5(包含)到10(包含)之间。

if双选结构

语法

执行条件表达式，如果结果是true，执行代码块1，否则，就执行代码块2

随堂练习

  1. 模拟用户登录操作（用户名和密码都用字符串），需要判断出登录成功和登录失败的情况。

if多选结构

语法

当表达式1为true时，执行代码块1

否则，如果表达式2为true，执行代码块2

否则，如果表达式3为true，执行代码块3

……

否则，执行代码块n

else if可以有任意多个

总结

  1. 在使用 if时，可以跟上0-多个else if，可以跟上0-1个else
  2. 当if下面的代码块只有一行的时候，可以省略大括号，不建议省略
  3. 在if语句的代码块内还可以再写if语句
  4. 在if的大括号中声明的变量只能在这个大括号内使用。在外面声明的变量可以在if中使用
  5. 能用else if和else的地方，不要用if

随堂练习

  1. 判断一个学生的成绩，如果大于100分，成绩不合法。如果90（包含）100（包含）为优秀，70（包含）90为良好，60（包含）~70为及格，60分以下输出不及格。

  1. 输入一个整数month代表月份，根据月份输出对应的季节。

    春季：3、4、5 夏季：6、7、8 秋季：9、10、11 冬季：12、1、2

  1. 输入工作年限(year)和月薪(money)，然后计算出应得的年终奖。

    工作不满1年（不包含）： 发月薪的1倍月薪年终奖；如果月薪大于8000, 那么就是发1.2倍月薪年终奖。

    工作不满3年（不包含）：发月薪的2倍月薪年终奖；如果月薪大于15000, 那么就是发3倍月薪年终奖。

    工作满3年（包含）以上：发月薪的3倍月薪年终奖；如果月薪大于20000, 那么就是发4倍月薪年终奖。

    闰年的计算方法：年数能被4整除，并且不能被 100整除；或者能被400整除的整数年份。

  1. 输入时（hour）、分（minute）、秒（second）的一个具体时间，要求打印出它的下一秒出来（一天24小时）。

    例如输入的是23时59分59秒，则输出00:00:00；例如输入17时09分59秒，则输出17:10:00。

switch结构

switch结构也是用于一些逻辑上的判断，但是应用场景与 if 不太一样。

语法

首先指定switch要判断的变量，之后进行值的匹配

如果变量是值1，就从case 值1开始往下执行

如果变量是值2，就从case值2开始往下执行

…..

如果上面都没有匹配到，就执行default

注意

  1. 当case匹配到值之后，程序会从这个case处开始往下执行，一直执行到第一个break为止。因此在使用switch时，注意大多数情况都不要遗漏break

  2. switch中的变量只能是byte、short、int、char、String和枚举，case后面的取值数据类型必须要跟变量一致

  3. switch中所有的case共用一个命名空间，多个case中不能声明同名变量，如果 想声明同名变量，可以使用 {} 包起来，作为一个代码块，代码块之间的命名空间是相互独立的。

    注意：这里的大括号不是switch中的语法，是代码块

    随堂练习

    1. 输入一个整数，对应的显示出星期几，例如: 输入“1”，则输出为“星期一”
    1. 接收一个人的成绩，如果成绩为: 90(包含)到100(包含)输出优秀，70(包含)到90输出良好，60(包含)到70输出及格，60分以下输出不及格。
    1. 查询水果的价格，根据输入水果(fruit)的名字，输出对应的水果的价格，例如苹果6块/斤，香蕉3元/斤，榴莲20元/斤，西瓜0.8元/斤。

if和switch应用场景

  1. 区间、范围判断时，使用if
  2. 对于某个变量的某些精确的取值判断时，可以使用switch，也可以使用if
  3. 如果取值过多，建议使用if，或者使用 策略设计模式

循环结构

循环结构就是在满足某个条件的前提下，不停地执行某一块代码，直到循环条件不满足为止

for循环结构

语法

  1. 执行初始化表达式，在初始化表达式中可以声明变量，这里声明的变量只能在for循环中才能使用
  2. 执行条件表达式，如果为true，就执行内部的代码块，如果为false，就终止循环
  3. 内部代码块执行完毕后，执行循环后表达式，之后从第 2 步继续执行

注意：

  1. 要循环n次，如果声明的变量从0开始，那么条件表达式只需要判断 这个变量 小于n即可。如果从1开始，需要判断这个变量小于等于n
  2. 初始化 表达式只会执行一次，因此可以在这里声明变量
  3. for中的三个表达式都可以进行省略
  4. for循环括号和大括号内声明的变量，只能在整个for循环中使用，在外面不能使用

随堂练习

  1. 输出0(包含)到100(包含)之间的数, 分别以递增和递减的方式实现；

  2. 输出1到100之间的奇数和偶数。

  3. 输入一个正整数n，计算1+2+3+…+n的和；

  4. 输入一个正整数n，计算1-2+3-4+5-6+…-(n-1)+n的和；

  5. 输出1到1000之间既能被5整除又能被3整除的数，并且每行输出5个。

  6. 求100到999之间的水仙花数。水仙花数的每个位上的数字的3次幂之和等于它本身（例如：1^3 + 5^3+ 3^3 = 153）。

  7. 编程找出四位整数abcd中满足下述关系的数，(ab+cd)*(ab+cd)=abcd（例如：(20 + 25) * (20 + 25)= 2025）。

while 循环

while一般用于不确定循环次数的场景

首先会判断条件表达式，如果条件表达式为true，就会执行代码块。之后再判断条件表达式，重复以上步骤，直到条件表达式为false为止

随堂练习

  1. 输入一个正整数n，计算1-2+3-4+5-6+…-(n-1)+n的和；

  2. 输出1到1000之间既能被5整除又能被3整除的数，并且每行输出5个。

for循环可以实现的需求，while循环同样可以实现，反之也一样

一般认为：能够确定循环次数的情况下，使用for循环。不能确定循环次数，或者想让循环不停下来的情况下，使用while循环

do…while循环

当想让循环体内代码至少执行一次时，使用do…while。

do…while循环最大的特点是先执行循环体，之后再判断是否满足条件表达式

总结

  1. 当能够确定循环次数时，使用for循环
  2. 当无法确定循环次数，或者想让循环一直执行时，使用while循环
  3. 当想让循环体至少会执行一次时，使用 do….while循环
  4. 大括号内定义的变量只在大括号内生效，外部无法使用
  5. 当循环体只有 一行代码时，for循环和while循环的大括号可以省略，但不建议
  6. while和for循环小括号后面注意 一定不能 加分号

方法

方法介绍

方法是定义在 类 中的一小段功能模块，方法是一段独立的小程序，可以被其他的代码调用。在有些语言中也会称之为 函数，或者过程

为什么要使用方法？为什么要定义方法？

  1. 把某一段代码抽取成一个方法，这个方法就是个独立的小功能，这个功能可以被其他的功能模块直接使用，不用重新再写一遍。比如淘宝在下单、查看订单、支付、查看物流、加入购物车等功能都需要判断用户是否登录，判断是否登录的功能就可以抽取成方法，其他模块需要判断是否登录时，直接使用这个功能即可，不需要再写一遍。提高了代码的复用性
  2. 之前的程序，把所有的代码全部写到了main方法中，当需求较为复杂时，可能会导致代码量非常庞大，此时对于阅读和维护都是极其不便的。我们可以根据功能模块将代码拆分成不同的方法，对于每个方法而言都是一个独立的小程序，比如A方法用于判断用户是否登录，B方法用于判断年份是否是闰年，C方法获取每个月的天数，每个方法的代码量都会比较小，阅读和维护就比较方便，提高了代码的 可读性 以及 可维护性

方法的声明与调用

语法

  1. 修饰符：用来控制该方法的访问权限，目前全部写成 public，在面向对象章节之前，还要加一个 static

  2. 返回值类型：方法的调用就像是使用运算符进行计算，运算符计算完毕后需要提供一个计算结果，在前面定义一个变量进行接收。方法执行完之后也可以提供一个 “计算结果”，在前面定义一个变量进行接收，返回值类型就是在定义这个计算结果是什么数据类型。如果方法不需要提供给调用者任何的数据，返回值类型需要定义为 void

  3. 方法名：给方法进行命名，在使用方法时通过方法名进行调用。方法名的定义要遵循标识符规则，并且还要遵循小写字母开头的驼峰规则

  4. 参数列表（也叫做形式参数，或者形参）：在定义方法时，有些不确定的参数需要调用者进行传递，此时需要定义在参数列表中。比如定义一个方法用于判断一个年份是否是闰年，年份应该由调用者提供，年份就可以定义成参数。参数列表声明规则

    允许定义多个

  5. 方法体内容：定义该方法的具体功能逻辑

  6. 返回值：返回值就是这个方法执行完毕后的“计算结果”，提供给调用者。返回值数据类型需要与方法上定义的返回值类型一致。如果返回值类型为void，返回值可以省略，可以直接写 return; 或者return也可以省略。当方法执行遇到return关键字，方法调用就立即结束。如果方法定义了返回值类型，需要保证在所有可能发生的情况下方法都必须能返回一个数据

  7. main方法就是一个特殊的方法，它有固定的写法，由虚拟机调用

  8. 在A方法中定义的变量不能直接在B方法中使用

方法在声明之后是一个独立的功能模块，无法自己运行，只有当被其他代码调用时才能执行方法体的代码

  1. 方法名：需要与 方法声明时的方法名一致
  2. 参数列表（实际参数，实参）：传递方法调用时需要的数据。需要与方法定义时参数列表的数据类型、参数个数、参数顺序完全一致。参数名不需要一致。实参传递时不需要再定义数据类型
  3. 数据类型和变量名：接收方法调用完毕后的返回值，数据类型需要与方法定义时的返回值类型一致。如果返回值类型为void，就不需要定义变量接收。如果返回值类型不是void，也可以不接收，也可以接收

方法运用思路

  1. 分析功能，提取出需要编写到方法内的逻辑
  2. 确定修饰符，目前全部是public static
  3. 确定返回值类型，根据需求分析出该方法是否需要为调用者提供数据，比如需要返回订单状态等。如果需要为调用者提供数据，就根据数据的类别决定数据类型，如果不需要提供数据，就是void
  4. 确定方法名，见名知义
  5. 确定参数列表，根据需求分析出该方法是否存在不确定的变量，或者不知道取值的变量，这类变量就需要定义成参数列表让调用者使用时进行传递
  6. 确定返回值。如果是void返回值的方法，可以没有return，也可以直接 return;

注意点

在switch和循环中，如果遇到了return，同样会结束方法的运行，在switch中就不会继续找第一个break，而循环会立即终止

方法定义上面要求使用文档注释进行描述，描述出方法的功能、返回值含义、参数含义

方法运行流程

首先先抛出问题：定义一个方法，交换两个数的值

栈内存是一个使用栈数据结构实现的一个内存空间，特点是 先进后出、后进先出

当一个方法被调用时，会在当前线程的 栈内存 中创建一个 栈帧，这个栈帧记录着当前被调用的方法的信息，创建完毕后，栈帧就会压栈。之后，该方法内创建的变量，都会存到这个栈帧的空间中。 当方法被调用时，调用者会将参数以 值传递 的方式传递给被调用的方法， 当方法调用完之后，栈帧会弹栈并销毁。

因此，这里交换的两个变量其实是swap这个栈帧中的num1和num2，而main方法中的num1和num2是不会受到影响的。

方法的重载

在同一个类中，方法名相同，但参数列表不同的方法叫做重载方法

参数列表不同指的是参数的数据类型或者参数个数不同，而参数名不同不是方法重载

方法的重载可以让我们把相同功能的方法定义成一个名字，依据传递参数的不同自动去匹配到合适的方法，省去了大量的方法名记忆，避免了命名空间的污染

  • 不在同一个类中，不是重载
  • 参数名不同，不是重载
  • 返回值类型不同，不是重载
  • 修饰符不同，不是重载
