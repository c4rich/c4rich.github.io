面相对象

面向对象概念

面向过程与面向对象

编程语言大体上可以分为两类：面向过程、面向对象。

  • 面向过程
    • 代表语言：C语言
    • 开发思想：先做什么，后做什么
    • 特点：性能比面向对象略高，适合做一些小系统，比如计算器、门禁系统、红绿灯
    • 封装：封装函数
  • 面向对象
    • 代表语言：Java，C++、C#等
    • 开发思想：强调宏观的把控，微观上依然是面向过程
    • 特点：性能略低，但代码的可读性、可维护性极高，适合做大型的商业级应用，比如管理系统、网站、APP、小程序等。
    • 封装：封装方法、类、数据

类和对象

对象：万物皆对象，具体到一个特定的事物都是一个对象，比如一台电脑、一张桌子、一名学生

类：将现实中一些相似、相同的事物 抽取出公共的特点就是 类。比如：电脑、桌子、学生。类中共有的特点，比如电脑有内存、硬盘、显示器分辨率等参数的取值，在类中是无法确定的，因为没有具体到某个事物，只有 具体到某个事物时，才能够确定这些参数，具体到的这台电脑就是对象。类像是一个模板，模板中将这类事物的特点进行了抽取，而这些属性具体的参数并没有提供，需要对象进行填充

面向对象编程

类的定义

类的定义使用 class 关键字

在类中定义的 非static修饰的 变量叫做 成员变量，非static修饰的 方法叫做 成员方法。

注意

  1. 在一个Java文件中允许定义多个类，public 修饰的类最多只能有一个，并且public修饰的类名必须与文件名一致
  2. 类名命名规范：允许使用数字、字母、下划线、美元符号组成，并且首字母不能是数字。一般只用 数字和字母，并且遵循大驼峰规则：每个单词的首字母大写，并且第一个字母首字母大写
  3. 在类中允许定义变量，不使用static修饰叫做 成员变量。成员变量在定义时可以不去初始化，成员变量有默认值，取值方式与数组一样：整数默认0，浮点数默认0.0，字符默认\u0000 ，布尔默认 false，引用类型默认null
  4. 在类中允许定义方法，不使用static修饰叫做 成员方法。成员方法中可以使用成员变量
  5. 方法中参数列表可以是引用类型，因此参数也可以传递一个对象

成员变量和局部变量区别

  • 定义位置不同
    • 成员变量 定义在类中
    • 局部变量定义在代码块中
  • 所属不同
    • 成员变量属于对象
    • 局部变量 属于代码块
  • 内存中存储位置不同
    • 成员变量存储在堆内存
    • 局部变量存储在栈内存
  • 生命周期不同
    • 成员变量随着对象的创建而创建，随着对象的销毁而销毁
    • 局部变量随着代码块的进入而创建，随着代码块的执行完毕而销毁
  • 修饰符不同
    • 成员变量允许有 public、protected、默认、private、static、final 修饰符
    • 局部变量只能有final
  • 初始化不同
    • 成员变量默认有初始化值
    • 局部变量没有初始化值

在JVM中有一个线程叫做 GC线程，也叫做垃圾回收线程，它的作用是回收堆内存中没有用的对象。在程序运行过程中，GC线程会 不定时地扫描堆内存中的对象，在扫描过程中，如果发现某个对象已经没有程序在使用了，就会在某个时间将这个对象进行回收（销毁）

匿名对象

在我们使用new关键字创建对象时，new 类名() 才是我们创建出的对象，为了方便对对象进行操作，一般定义一个变量接收这个对象的引用，给这个变量进行命名，后续的操作使用对象名进行操作

有时候我们可能 仅仅是想调用这个对象内的某个方法，此时也可以不定义变量名进行接收，这种没有定义变量名接收的就叫做匿名对象

一般在只想调用对象的某个方法时，可以使用匿名对象，或者在方法传参时可以使用匿名对象

构造方法

构造方法简介

构造方法又叫做 构造器，作用是在创建对象时完成一些初始化操作，构造方法是在创建对象时进行调用，使用 new 关键字。前面提到的创建对象的语法，实际上new的是这个类的构造方法

构造方法声明语法与普通的方法不一样。构造方法没有返回值，方法名必须要与类名一致

  1. 修饰符：控制构造方法的访问权限
  2. 方法名：构造方法的方法名 必须是类名
  3. 参数列表：构造方法的参数列表与方法的参数列表规则一样。一般构造方法的作用是创建对象时完成一些初始化操作，因此参数列表一般用于传递一些用来初始化使用的参数
  4. 代码块：构造方法被调用时执行的代码块
  5. 构造方法定义在类中，与成员变量和成员方法是平级

构造方法使用

无参构造方法

一般简称为无参构造，是没有参数的构造方法。当一个类没有构造方法时，编译器会默认给我们提供一个无参构造，也可以自己定义一个无参构造。

有参构造方法

一般简称为有参构造，默认情况下没有有参构造，开发者可以自己定义有参构造。当一个类中有了有参构造之后，就没有默认的无参构造了，如果需要使用无参构造，也可以自己定义。建议每一个类中都要定义一个无参构造

构造方法是可以重载的

问题：创建对象完全是由构造方法完成的吗？

不是。创建对象分为以下几步

  1. 直接在堆内存中开辟存储对象的区域，并且完成成员变量默认的初始化
  2. 执行对象成员变量的显式初始化
  3. 执行构造方法
  4. 返回该对象在堆内存中的引用

调用构造方法仅仅是对象创建的其中一步，调用了构造方法可能并不一定创建了对象

构造方法和成员方法的区别

  • 定义格式不同
    • 构造方法没有返回值，方法名必须与类名一致
    • 成员方法可以有返回值，方法名可以自定义
  • 调用时机不同
    • 构造方法在创建对象时调用
    • 成员方法在创建对象后调用
  • 调用方式不同
    • 构造方法使用new关键字调用
    • 成员方法使用对象.方法名调用
  • 调用次数区别
    • 构造方法使用new关键字只能调用一次，使用this关键字可以在构造方法中进行调用
    • 成员方法在创建完对象后可以被调用任意次。

this关键字

简单介绍

在方法中使用变量时，遵循了就近原则。当方法参数和成员变量存在同名变量时，如果直接使用变量名来进行操作，会遵循就近原则，即：使用的是方法的参数。

而现在想要让程序把构造方法接收到的name赋值给当前对象的name成员变量，我们可以使用 对象名.变量名 进行区分。但是，在构造方法中并没有对象名，因此JVM在创建对象后会为我们默认分配一个引用 this，这个this就代表 当前对象

this关键字调用成员变量

可以在类中使用 this.成员变量名 调用 当前对象的成员变量。当方法与成员变量的变量名存在重复时，编译器会采用就近原则，此时可以使用 this 关键字进行区分。当不存在重复时，this可以省略，编译器帮我们把this加上。

this关键字调用成员方法

this关键字可以用来调用成员方法，默认情况下也是可以省略，编译器为我们加上this关键字。this关键字可以用来区分一个方法是否带有static修饰

由于方法的调用一般不存在歧义，因此this关键字调用方法时，可以省略。

static关键字

被static关键字修饰的变量和方法叫做 静态变量 和 静态方法，static关键字修饰的不属于对象，而是属于类。

静态变量

给变量加一个static关键字就是静态变量

注意

  1. 一个类中定义的static变量是属于类的，并且全局只有一份。
  2. 静态变量可以用 对象.变量名 调用，也可以用 类名.变量名 调用，一般建议使用类名

在首次使用一个类时，这个类的信息会被加载到 方法区（永久代/共享数据区/元空间），之后再使用这个类，不会重复加载，即一个类在方法区里只会有一份。而静态变量和静态方法则是跟随着这个类存储到了方法区，不管使用对象名还是类名操作静态变量，实际上都是在操作方法区中的一份class信息

通过画图分析static变量，我们能够推断出，在开发中应当尽可能避免修改静态变量的值

我们可以给静态变量加一个 final 关键字，定义成常量，让其他的程序无法对其进行修改。

对于静态的常量，我们一般会单独创建一个类进行管理

静态方法

静态方法是使用static关键字进行修饰，修饰的方法就叫做 静态方法

静态方法同样是属于类，调用时可以使用对象名调用，也可以使用类名调用，一般建议用类名

为什么要用静态方法？

比如Arrays、Math两个类中几乎所有的方法都是静态方法

对于这两个类，它们提供的所有的方法都是 工具性质的 方法，我们其实只需要能够保证这些方法可以被正常调用即可，至于调用方式并不关心，此时不管创建多少对象都是无意义的，反而会造成堆内存空间的浪费，因此我们使用到方法区的特性：同一个类只会被加载一次，因此将工具性质的方法放到方法区内，不管创建多少对象，方法只有一份。为了避免工具性质的方法所在的类重复创建对象，一般会将无参构造私有化，让外面的代码无法创建对象。

工具方法所在的类也可以叫 工具类

static关键字之间的调用

当类被加载时，对象不一定被创建，因此静态方法不能调用非静态的变量和方法。而成员方法属于对象，当调用成员方法时对象一定被创建了，类也一定被加载了，因此此时属于类的数据和方法都可以调用，即非静态的方法能够调用 静态的变量和方法

总结起来一句话：**static修饰的只能使用static修饰的，非static无限制 **

静态代码块

静态代码块

使用static关键字修饰，定义在类中与静态变量和静态方法平级的代码块叫做静态代码块，作用是给类做初始化操作

静态代码块会随着类的加载只执行一次，如果有多个静态代码块，会按照从上往下的顺序进行执行。

构造代码块

构造代码块是定义在类中，与成员变量和成员方法平级的，不使用static修饰的代码块，主要是给对象进行初始化操作

构造代码块会随着new的时候执行一次，下次new的时候，还会执行一次

执行顺序：静态代码块->构造代码块->构造方法

package 包

package概述

包就像是文件夹的一样的东西，作用是对类进行管理和分类

对于同名类，也可以使用包进行管理，不同包下允许同名类

包的定义

命名：包名一般采用公司域名倒着写，遵循标识符规范，一般全部字母小写，单词间使用 . 。包名不能以 java 开头

定义到某个包下的类，需要在类最上面第一行代码声明所在包，使用 package 关键字

Java常用包

java中的常用包 说明
java.lang 包含一些Java语言的核心类，如String、Math、System等。lang包下的类使用时不需要导包
java.awt 包含了构成抽象窗口工具集（abstract window toolkits）的多个类，这些类被用来构建和管理应用程序的图形用户界面(GUI)。
java.net 包含执行与网络相关的操作的类。
java.io 包含能提供多种输入/输出功能的类。
java.util 包含一些实用工具类，如定义系统特性、使用与日期日历相关的函数。

Java类的访问

当我们定义了一个类之后，可以在其他代码中进行访问，访问时需要导包，导包使用 import

当需要使用两个不同包下的同名类时，其中一个可以导包，另外一个只能使用 全类名

当使用的类就在当前包下，导包可以省略
