封装

什么是封装

面向对象三大特性：继承、封装、多态

封装就是隐藏内部的细节，对外暴露公共的访问方式

封装的好处：提高代码的复用性、提高代码的可维护性、降低开发成本

在我们定义一些类时，某些成员变量对于取值有范围要求，比如年龄不能为负数，一般也不会超过150。而前面的代码中对成员变量并没有进行封装，而是直接暴露在外，让程序直接使用 对象.变量名 的方式进行访问，这样就会存在年龄赋值成负数的情况

为了避免某些成员变量取了非法取值的情况，我们一般会对成员变量进行封装。要求：私有化成员变量，提供 公共的访问方法

setXxx、getXxx方法一般称之为get、set方法，也可以称之为getter、setter，它们的作用就是给成员变量赋值以及获取成员变量。get、set方法命名方式有特殊要求：要求必须以get、set开头，后面紧跟着首字母大写的成员变量名

建议：不管成员变量是否有取值范围要求，都要对成员变量进行私有，并提供公共的get、set方法

成员变量和属性的区别

定义在类中的变量严格来说叫做成员变量，不叫做属性。

属性指的是：getXxx、setXxx

当一个类中有name变量，而没有getName和setName时，我们认为，这类中没有name属性

当一个类中没有name变量，而有getName和setName时，我们认为，这类中有name属性

程序中的继承指的是一个类继承另一个类，继承后，子类就拥有了父类的所有方法和变量。

A类继承B类，B类一般称之为 父类（超类/基类），A类一般称之为 子类

方法的重写

当父类中的某个方法不太满足子类的需求时，可以在子类中对父类的这个方法进行 重写，也叫做方法的 覆盖。方法重写要求方法名和参数列表与父类方法 完全一致

方法重写的好处：

  1. 可以在不改变现有逻辑或者测试代码的情况下，对某个功能进行改写
  2. 可以让子类和父类共用一个方法名，为后面的多态打基础
  3. 可以提高代码的扩展性和可维护性
  4. @Override注解一般标识在重写方法上，标识该方法是重写自父类的方法，如果该方法不是重写的方法，编译会报错
  5. 方法重写时，子类重写方法的修饰符权限要大于等于父类。一般会与父类保持一致
  6. 方法重写时，子类方法的返回值要小于等于父类方法。一般会与父类保持一致。
  7. 当调用一个对象的某个成员方法时，会首先从该对象 对应的类中找这个方法，如果找不到，就从该类的父类进行寻找。如果在某个类中找到了该方法，就不会再从父类中寻找，而是直接执行这个类里的方法

super关键字

super关键字的作用是调用 父类存储空间 的方法和变量，super关键字使用方式与this几乎一样。

super关键字调用构造方法

当new一个子类时，这个子类的构造方法默认 会调用父类的空参构造。因为子类中有些成员变量是从父类中继承而来的，这些变量在创建对象时可能需要进行初始化，这些变量的初始化需要在父类的构造方法中进行，因此创建子类对象时，必须要调用父类的构造方法，默认调用的是父类的空参构造。

初始化子类必须要先初始化父类

注意：super关键字仅仅是调用了父类的构造方法，并没有创建父类对象

父类存储空间概念

成员变量不存在重写（不能被覆盖），当子类和父类都定义一个name变量时，这两个name是不同的两个变量

当创建一个对象，对象被存储到堆内存时，会根据继承关系划分空间。

final关键字

  1. final关键字可以使用在变量上，final修饰的变量会视为常量，当赋值一次后就不能再进行改变。
  2. final可以使用在方法上，被final修饰的方法不能被重写。
  3. final可以使用在类上，被final修饰的类不能被继承
  4. final修改引用类型变量，不能改变变量的地址，但是可以修改变量在堆内存中的值

final关键字修饰类，一般是为了防止某些类有独有的特性，不让其他类能够对这个特性进行改写。比如String在使用时可以不用new，并且String内部有字符串常量池，如果String 能够被继承，这两个特性可能就会被篡改，因此String使用final关键字修饰。

同时，对于工具类，我们只是想提供一些工具性质的方法，并不希望其他类能够对其进行改写或者扩展，一般也可以使用final关键字修饰

Object类

介绍

在Java中所有的类都默认直接或间接地继承了Object类，Object类中提供了一些常用的公共的方法，其他的类因此也拥有这些方法。此外，Object还会应用到多态上，我们可以让一个方法接收Object类型的参数，这样该方法就能传递所有类型的数据。

Object内部定义了一些常用方法，本章暂时只介绍 equals、toString、hashCode

toString方法

toString方法的作用是返回一个字符串，一般认为是将当前对象按照特定格式转换成一个字符串

toString默认输出的是 全类名@十六进制hashCode，一般情况下一个类会重写toString，重写成返回当前类对象的所有成员变量值

System.out.println方法如果输出一个 对象，默认输出的是它的toString方法，因此如果想让一个对象能够被直接输出成员变量，一般都需要重写它的toString

hashCode和equals

hashCode 是经过哈希算法用一个数字来表示一个对象。hashCode方法一般与equals连用的。

equals方法默认比较的是两个对象地址是否相同，一般会重写成比较内容是否相同

在开发中并不一定会比较所有的成员变量，比如如果身份证号已经相同了，其余的成员变量就不用比较了，因为相同身份证肯定是同一个人

如果两个对象equals相同，我们认为hashCode一定相同，如果重写了equals方法，可能只会比较一个对象的某几个成员变量，而hashCode转换时是依据所有成员变量进行转换，因此如果重写了equals，我们一般还会重写hashCode，equals比较哪些成员变量，hashCode就也比较哪些成员变量

注意

  1. 如果两个对象的equals相同，我们认为hashCode也相同
  2. 如果两个对象的hashCode相同，equals不一定相同。

概述

多态指的是对于同一个方法，由于对象的不同，调用的结果也不同

多态的使用条件

  1. 必须在继承中，并且子类重写了父类的方法
  2. 必须是父类引用指向子类对象。

  1. 方法参数如果是父类，调用者传参时可以传递父类对象也可以传递它的任意子类对象
  2. 方法返回值类型如果是父类，返回值可以是父类对象也可以是它的任意子类对象。

多态的好处

  1. 提高了程序的扩展性。可以在不改变原有代码的情况下，通过继承+重写的方式对某个方法的功能进行扩展。
  2. 提高了程序的可维护性。不需要关注某个流程的具体实现，只需要关注子类中重写的某个方法即可

总结

  • 向上转型
    • 优点：本质上是多态的体现，能够提高程序的扩展性、可维护性等。
    • 缺点：无法使用子类自己定义的独有方法
  • 向下转型
    • 优点：能够使用子类独有的方法
    • 缺点：使用不当可能会存在 ClassCastException（类型转换异常），因此在使用向下转型之前，需要使用 instanceof 关键字先判断是否是这个类型，再进行强转。

多态中成员方法和成员变量的特点

成员方法：方法存在着重写机制，当调用一个对象的方法时，会先从这个对象的类中找这个方法，如果能找到，就调用该类中该方法，找不到则去父类中找，直到找到为止。编译看左边，运行看右边。

成员变量：成员变量不存在重写，当父类和子类都定义了同名变量时，这两个是不同的变量，在堆内存中存储到不同的存储空间中。当直接调用成员变量时，不管是编译还是运行都是看左边，左边是什么类，调用的就是这个类中的变量。

抽象类

抽象类概述

在某些情况下，我们可能只知道某个方法作用是什么，但是具体如何实现并不知道。比如：计算一个图形的面积。方法的名称和返回值我们能够定义，但是没有具体到什么图形，我们并不知道它的面积如何计算。此时，这个计算面积的方法就需要定义成 抽象方法，而这个方法所在的类，就需要定义成 抽象类

抽象类使用

抽象方法用于定义这些暂时不知道实现方式的方法，这些方法要求 子类进行实现。非抽象的子类必须要重写父类的抽象方法。

注意

  1. 抽象方法和抽象类都需要使用 abstract 关键字进行修饰，其中，抽象方法不需要方法体
  2. 只要一个类中包含了抽象方法，这个类必须得是抽象类
  3. 抽象类允许有构造方法和成员变量，抽象类不能创建对象。
  4. 抽象类依然支持多态
  5. 抽象方法必须得能够被子类重写，因此不能使用private修饰，建议抽象方法全部使用public
  6. 抽象方法不能用final和static修饰，因为这两个关键字修饰的方法都不能被重写

接口

接口的概述

接口是比抽象类更加抽象的一个 “类”。

接口的特点如下

  1. 接口声明使用 interface 关键字
  2. 接口中所有的方法全部都是抽象方法，并且默认且只能使用 public abstract 修饰（JDK7及之前），JDK8开始允许使用 default和static 修饰
  3. 接口同样适用于多态

接口的使用

接口的作用有两种

  1. 作为一种“规范”，事先在接口中定义好某个模块需要哪些功能，之后，交给子类进行实现
  1. 作为一种功能上的扩展，给已经完善的某个类再扩展出某个功能

接口定义完毕后必须要有“子类”进行“继承”。接口的“子类”我们一般称之为实现类，实现类“继承”接口的行为我们一般称之为实现一个接口，使用关键字是 implements，实现类重写接口的抽象方法我们一般称之为 实现某个方法

第二种用法：作为功能的扩展

接口应用于多态

接口和抽象类的区别

抽象类一般作为一个模板进行使用。当一个流程多数子流程都是可以确定的，但是某个流程无法确定时，可以使用抽象类。这里也叫做 模板方法设计模式

而接口一般作为规范或者功能的扩展来使用。

接口的继承

接口之间是可以继承的，并且是可以多继承

一个类允许实现多个接口。一般一个类实现的多个接口中最好不要有同名方法，这个要求不是语法规定，但是阅读代码时容易有歧义

一个接口允许继承多个接口

接口的默认方法

JDK8中接口允许有默认方法，这个方法是允许有方法体的。在开发中可能会存在要增加接口方法或者要更新JDK版本的情况，此时接口如果增加了新的抽象方法，这个接口所有的实现类都需要重写这个方法，此时工作量巨大，并且实际中并不是所有的类都需要实现新的方法，此时就可以使用接口的默认方法

接口的静态方法

JDK8之后，接口允许有静态方法，接口的静态方法与普通的类中的静态方法规则是一致的，public可以省略

接口中的变量

接口中允许定义变量，这些变量默认是被 public static final 修饰的静态常量

使用内部类的场景

  1. 在描述一个事物时，内部还有其他的事物
  2. 在一个类中需要使用到某个类，但是这个类不想让其他类能访问
