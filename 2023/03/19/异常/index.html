异常概述

什么是异常

程序在编写和运行的过程中，可能会出现一些问题，比如操作数组时可能会超出数组的有效索引范围，或者使用了null去调用方法和变量，这些情况下都会抛出对应的异常

异常体系

继承关系

Java中异常也是一个类 Exception，它的父类是 Throwable，Throwable还有一个子类 Error

错误：错误指的是程序运行过程中由于不合理的代码而发生的一些较为严重的、开发者无法解决的问题。错误只能 尽可能在代码、调优等方面去进行避免

异常：异常指程序运行过程中，由于场景没完全考虑等方面造成的一些问题，异常能够通过编码去进行解决

Throwable提供了4个常用的方法

方法名 描述
public String getMessage() 返回此throwable的详细消息字符串。
public String toString() 返回此 throwable 的简短描述
public void printStackTrace() 打印异常的堆栈的跟踪信息
public synchronized Throwable getCause() 获取异常文本中提示的可能出现异常的原因

Exception介绍

Exception是Java中异常的父类，Java中异常分为两种：编译时异常（Exception）、运行时异常（RuntimeException）

编译时异常：在编译时就进行检查，如果这里的代码抛出了编译时异常，必须在代码中进行捕获，或者向上抛出

运行时异常：在运行时才有可能出现的异常，编译时不进行检查，程序中抛出了运行时异常可以不对其进行处理

建议两种异常都进行处理。

抛出异常

在程序运行的过程中，如果出现了异常，程序会将异常 抛出，异常一旦进行抛出，程序就必须对这个异常进行处理，如果程序没有进行处理，就让这个方法的调用者进行处理，直到交给虚拟机进行处理

在开发中可以主动抛出异常，抛出一个异常之后，程序能够在后续的执行中捕获到异常，可以进行一些特殊的处理。因此我们如果想要给用户、其他程序提示一些文本内容，应该使用异常进行提示

抛出异常使用 throw 关键字，当throw抛出的是运行时异常时，方法上不需要做其他处理，当抛出的是编译时异常时，方法上需要将这个异常声明出来，声明一个异常使用 throws

在一个方法中抛出一个异常，如果这个方法没有对该异常进行捕获，该方法的运行就立即结束

声明异常

当我们抛出一个异常之后，在该方法或者该方法的调用者内部就需要对这个异常进行处理，为了让调用者能够知道需要处理哪些异常，我们需要将一个方法中可能会抛出的异常 声明 到方法上

捕获异常

当一个方法抛出了异常，这个方法的调用者要么把这个异常继续抛给上一层，要么在调用者内部对这个异常进行处理。

捕获异常使用 try...catch...finally 语法，其中try是必须有，catch和finally至少要有其中一个，catch可以有多个

代码示例

  1. 当try中的代码发生了异常之后，try中的程序立即停止运行，程序会跳到对应的catch中继续向后运行
  2. finally中的代码不管程序是否出现了异常，finally中代码都会执行，并且即使方法return了，finally也会执行
  3. try仅仅是验证代码是否有异常，不会进行异常的处理，而是交给下面的对应的catch处理。如果catch没有处理到该异常，就会继续向上抛给它的调用者。

随堂练习

1. 编写程序接收用户输入分数信息，如果分数在0—100之间，输出成绩。如果成绩不在该范围内，抛出异常信息，提示分数必须在0—100之间。

  1. 写一个方法void isTriangle(int a,int b,int c)，判断三个参数是否能构成一个三角形， 如果不能则抛出异常IllegalArgumentException，显示异常信息 “a,b,c不能构成三角形”，如果可以构成则显示三角形三个边长，在主方法中得到命令行输入的三个整数， 调用此方法，并捕获异常。
  1. 编写一个计算N个整数平均值的程序。程序应该提示用户输入N的值，然后必须输入所有N个数。如果用户输入的值是一个负数，则应该抛出一个异常并捕获，提示“N必须是正数或者0”。并提示用户再次输入该数

自定义异常

Java中提供了大量的异常，这些异常根据实际场景决定使用哪一个。这些异常的作用是使异常能够见名知义。实际开发中这些异常可能并不能满足我们的需求，此时就需要自定义一个异常

自定义异常可以继承Java中的任意一个异常，一般会继承Exception或者RuntimeException

方法重写异常

方法重写时，如果父类方法中声明了异常，子类方法可以不声明，如果声明，则必须是父类方法上的异常，或者是这个异常的子类

try-with-resource

当我们的程序与其他的程序建立连接时，此时这个连接一般称之为资源，当资源操作完毕后，需要手动进行关闭，JDK7以前关闭代码比较复杂

JDK7之后提供了 try-with-resource的语法

资源对象必须实现 AutoCloseable，当try…catch执行完毕之后，程序会自动帮我们关闭资源对象
